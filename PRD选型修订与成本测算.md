# PRD 选型修订与成本测算

> 配套文档:针对《AI 陪玩搭子 — 产品需求文档(PRD)v0.1》第 6 章技术选型 / 第 10.4 风险表的修订建议
> 编写日期:2026/5/19 · 价格基准:2026/5

---

## 一、必改项(4 条)

### 修订 1 — LLM 模型名【必改】

- **现状**:6.2 选型表 LLM 写 `DeepSeek-Chat`。
- **问题**:`deepseek-chat` 现仅为 DeepSeek V4 Flash 非思考模式的兼容别名,**官方计划 2026-07-24 弃用**,正撞产品上线期。
- **改为**:代码与文档统一写明确模型名(V4 Flash 系正式 model id),不使用 `deepseek-chat` 别名。

### 修订 2 — LLM 选型理由【必改】

- **现状**:选型理由写"中文聊天能力强"。
- **问题**:DeepSeek V4 公开 benchmark 集中在数学/代码/推理/长上下文/中文知识问答,属推理型模型;角色扮演、拟人撒娇、聊骚口语化等本产品核心维度**无公开评测覆盖**,且推理型模型易"书面化、爱说教"。
- **改为**:选型理由改为「成本基线,角色扮演/拟人对话表现需在 MVP 内 A/B 验证」。Week 1 用同一套角色 prompt 横向评测 DeepSeek V4 Flash / 通义 Qwen / 智谱 GLM / 豆包,评测维度:角色不崩、不说教、口语自然、情绪到位、聊骚尺度。

### 修订 3 — TTS 型号 + 新增成本测算【必改/必补】

- **现状**:6.2 写 MiniMax Speech-02(默认理解为 HD)。
- **改为**:MVP 用 **Speech-02 Turbo**($30/百万字符,比 HD 便宜 40%),陪伴场景音质足够;HD 留作未来高端付费档。
- **必补**:PRD 增加"单分钟对话成本测算"小节(见本文档第二节),并据此倒推时长包定价,确保毛利为正。

### 修订 4 — VAD 优先级上调【必改】

- **现状**:F15 VAD 静音检测列在 P1。
- **问题**:成本测算显示 ASR 按"录入时长"计费,是最大成本项(占 ~86%)。前端 VAD 只上传有效语音段,可砍掉 60-70% ASR 计费时长。
- **改为**:**F15 VAD 从 P1 上调到 P0**——它既是体验功能也是核心成本功能。

### 修订 5 — 架构决策点【必决策】

- **现状**:6.2 把"火山实时对话 AI SDK / OpenAI Realtime API"列为"可选"。
- **改为**:把"三段式 vs 实时一体化"提升为 **Week 1(Day 1-5)必须跑双 demo 对比并锁定**的决策事项。
  - 三段式:角色/游戏知识/聊骚尺度可控,但延迟难压(1.5s 目标偏乐观)。
  - 一体化:延迟 500-800ms 体验好,但端到端模型角色可控性弱、聊骚易被安全策略拦截。
  - 注意:本产品"聊骚/假恋爱"卖点与 10.2"敏感词过滤"存在张力,三段式因 LLM 可控更易调尺度——这应纳入架构选择考量,而非仅看延迟。

## 二、单分钟对话成本测算

### 已确认单价(2026/5)

| 项 | 单价 | 备注 |
|---|---|---|
| ASR 火山按时长后付费 | ¥0.08/分钟 | 每月前 1000 分钟免费;需 Day1-2 复核 |
| LLM V4 Flash 输入(缓存命中) | $0.0028/M token | 官方文档确认 |
| LLM V4 Flash 输入(缓存未命中) | $0.14/M token | 官方文档确认 |
| LLM V4 Flash 输出 | $0.28/M token | 官方文档确认 |
| TTS Speech-02 Turbo | $30/百万字符 | 计费单位"字符"需 Day1-2 复核 |
| TTS Speech-02 HD | $50/百万字符 | 备用高端档 |

汇率:$1 = ¥7.2

### 建模假设(每 1 分钟"对话时长")

- AI 实际输出语音 ≈ 12 秒 ≈ 60 个中文字 ≈ 90 LLM token(中文约 1.5 token/字)
- 每轮注入 system prompt(角色+游戏知识)≈ 800 token,命中缓存
- 历史摘要 + 本轮用户输入 ≈ 150 token,未命中缓存
- ASR 按整段对话时长计费(未做 VAD 优化场景)

### 每分钟对话成本

| 模块 | 用量 | 成本/分钟 | 占比 |
|---|---|---|---|
| ASR | 1 分钟 | ¥0.0800 | 86% |
| LLM 输入(缓存) | 800 token | ¥0.000016 | ~0% |
| LLM 输入(未缓存) | 150 token | ¥0.00015 | ~0% |
| LLM 输出 | 90 token | ¥0.00018 | ~0% |
| TTS Turbo | 60 字 | ¥0.0130 | 14% |
| **合计** | | **≈ ¥0.093/分钟** | 100% |

### 关键结论

1. **ASR 是最大成本项(~86%),不是 TTS。** PRD 10.4 风险表"TTS 成本超预算"的判断需修正——真正的成本大头是 ASR,因为麦克风全程开启,录入时长 ≈ 全部对话时长。
2. **LLM 成本可忽略**(三项合计 < 0.4%),前提是 system prompt 命中 prompt cache——把固定的角色人设+游戏知识放在 prompt 开头。
3. **VAD 是核心降本手段**:只上传有效语音段,可将 ASR 计费时长砍掉 60-70%,可变成本从 ¥0.093 降到约 ¥0.05/分钟。

## 三、时长包定价倒推

取 API 成本 ¥0.093/分钟 + 30% 余量(并发限流/转码/重试/汇率波动)≈ **¥0.12/分钟可变成本**。

| 时长包 | API 成本 | 建议售价 | 毛利率 |
|---|---|---|---|
| 60 分钟体验包 | ¥7.2 | ¥19.9 | 64% |
| 300 分钟标准包 | ¥36 | ¥69 | 48% |
| 1000 分钟畅玩包 | ¥120 | ¥199 | 40% |

- 10 分钟免费试用成本 ≈ ¥1.2/人;目标 50 试用用户总成本 ≈ ¥60。
- 叠加 VAD 优化后,各档毛利率可再提升约 20 个百分点。

## 四、Day 1-2 必须亲自复核的数字

WebFetch 无法抓取官方文档动态页面,以下单价来自搜索结果,申请 API key 时务必在控制台核实:

1. 火山 ASR 流式识别按时长后付费精确单价 + 免费额度规则。
2. MiniMax Speech-02 Turbo 计费单位:按"字符"还是"UTF-8 字节"——中文按字节会贵 3 倍,直接影响成本测算。
3. DeepSeek prompt cache 的命中条件与 TTL,确认 system prompt 能稳定命中。
